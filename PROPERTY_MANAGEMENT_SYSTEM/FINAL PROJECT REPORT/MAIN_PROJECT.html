ABSTRACT

The Project ’Property Management System’ is a system based on trading property. The purpose of this project is so a buyer can easily buy the property of his choice, similarly, a seller can sell his own property. The proposed system shows the system has been provided to various sellers and buyers. Where a buyer can purchase a property through a transaction with some of his information. To make the system user-friendly, only the necessary information is provided between the buyer and the seller.

INTRODUCTION

Technology is changing very fast at the present time. Everything in the world right now is digitalized. Everything is in our hands today in need of time. The 'Property Management System' is very up-to-date to meet the needs of the times. In the past, buying or selling a property required a lot of time and was a costly affair. It doesn't matter in the digital age. Any buyer can buy any property at his convenience. Similarly, a seller can easily sell his own property in less time. Moreover, in the past, buyers and sellers had to face many problems due to informational errors. But buyers and sellers can transact with ease through the property management system and through verified information. The 'Property Management System' database is shown below

CREATE TABLE  "BUYERS"

  • PROPERTY

CREATE TABLE  "PROPERTY"

   (        "PRO_REG_NUM" NUMBER(10,0),

        "PRO_NAME" VARCHAR2(100),

        "PRO_TYPE" VARCHAR2(100) NOT NULL ENABLE,

        "PRO_DES" VARCHAR2(100),

        "PRO_PRICE" NUMBER(18,0),

         CONSTRAINT "PROPERTY_PK" PRIMARY KEY ("PRO_REG_NUM") ENABLE,

  • SELLERS

CREATE TABLE  "SELLERS"

   (        "SELL_NID" NUMBER(10,0) NOT NULL ENABLE,

  • TRANSACTION

CREATE TABLE  "TRANSACTION"

   (        "TRA_ID" NUMBER(10,2) NOT NULL ENABLE,

        "BYR_NID" NUMBER(10,0),

        "PRO_REG_NUM" NUMBER(10,0),

        "SELL_NID" NUMBER(10,0),

        "PURCHASE" NUMBER(10,0),

        "TRA_DATE" DATE,

         CONSTRAINT "TRANSACTION_PK" PRIMARY KEY ("TRA_ID") ENABLE,

         CONSTRAINT "TRANSACTION_FK" FOREIGN KEY ("BYR_NID")

         REFERENCES  "BUYERS" ("BYR_NID") ENABLE,

         CONSTRAINT "TRANSACTION_FK3" FOREIGN KEY ("PRO_REG_NUM")

         REFERENCES  "PROPERTY" ("PRO_REG_NUM") ENABLE,

         CONSTRAINT "TRANSACTION_FK2" FOREIGN KEY ("SELL_NID")

         REFERENCES  "SELLERS" ("SELL_NID") ENABLE

   )

  • USER_INFO

CREATE TABLE  "USER_INFO"

        "USER_EMAIL" VARCHAR2(100),
